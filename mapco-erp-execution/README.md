# MAPCO ERP SYSTEM - CRUD + Realm Production Phase

نسخة تنفيذية Desktop ERP لشركة طباعة وتغليف وقوالب تكسير.

## ما الجديد في هذه المرحلة

- ربط التنقل بالكامل.
- CRUD فعلي للشحنات والمخزون والإنتاج والمستندات.
- CRUD جديد لقائمة الأصناف.
- CRUD جديد لمكتبة القوالب.
- حركات مخزون: إضافة / صرف / تسوية.
- بحث سريع داخل كل صفحة تشغيلية.
- صلاحيات مبدئية: Admin يمكنه التعديل والحذف، Employee للعرض والمتابعة.
- اختيار ملفات للمستندات والقوالب من الجهاز عبر Electron dialog.
- تصدير CSV للتقارير من قاعدة البيانات.
- Realm يعمل داخل Electron Main Process، والواجهة تتعامل معه عبر preload/contextBridge.

## التشغيل

```bash
npm install
npm run dev
```

ثم افتح Terminal ثاني:

```bash
npm run electron:dev
```

## بيانات الدخول

- admin
- employee

أي كلمة مرور مقبولة في هذه النسخة التجريبية.

## مكان تخزين Realm

يتم حفظ البيانات داخل مجلد userData الخاص بالتطبيق:

```text
%APPDATA%/mapco-erp-system/data/mapco-erp.realm
```

## ملاحظة مهمة

لو كانت عندك بيانات Realm قديمة من نسخة سابقة ولم تظهر الجداول الجديدة، احذف ملف Realm القديم من مجلد البيانات أو شغل:

```bash
npm run clean:data
```

ثم افتح التطبيق مرة أخرى.

## Company profile update

- تم إضافة جدول CompanyProfile داخل Realm.
- تم تخزين شعار مهابة الفن داخل قاعدة البيانات كـ logoDataUrl.
- اسم الشركة المخزن: مهابة الفن للدعاية والإعلان.
- إذا كانت لديك قاعدة بيانات قديمة ولم يظهر الشعار، شغّل: npm run clean:data ثم افتح البرنامج من جديد.

## آخر تحديث تنفيذي

تمت إضافة التعديلات التالية:

- تثبيت عرض الأرقام باللغة الإنجليزية في جميع القيم المالية والتواريخ والوقت قدر الإمكان.
- حفظ شعار واسم شركة مهابة الفن للدعاية والإعلان داخل قاعدة بيانات Realm.
- ربط أوامر الإنتاج بالمخزون: كل أمر إنتاج قالب يجب أن يحتوي على الأقل على خشب + سكين + ريجة + ربر.
- دعم إضافة مكونات اختيارية مثل البنشات وسكين السرسرة.
- عند حفظ أمر إنتاج مرتبط بالمخزون يتم صرف الكميات من المخزون وتسجيل حركة مخزون تلقائية باسم أمر الإنتاج.
- عند تعديل أو حذف أمر إنتاج يتم استرجاع حركة الصرف السابقة ثم إعادة تطبيق الحركة الجديدة حتى لا يحدث صرف مكرر.
- توسيع صفحة الشحنات لتشمل: رقم الفاتورة، PO، B/L، رقم الحاوية، الخط الملاحي، Incoterm، العملة، سعر الصرف، نسب الجمارك والضريبة، رابط فولدر Google Drive، وجدول مستندات لكل شحنة.
- تم الاستفادة من ملف Excel المرفق كفكرة تنظيمية لمركز متابعة الشحنات: رقم الشحنة، التاريخ، الحالة، زر الفتح/الرابط، والرسوم الحالية.

ملاحظة: لو كانت لديك بيانات قديمة من نسخة سابقة، شغل:

```bash
npm run clean:data
```

ثم شغل النظام من جديد حتى تظهر الحقول الجديدة والبيانات التجريبية الجديدة.

## Backup Manager / Restore

تمت إضافة مدير نسخ احتياطي احترافي داخل صفحة الإعدادات:

- إنشاء نسخة احتياطية كاملة من قاعدة بيانات Realm بامتداد `.mabackup`.
- إمكانية حفظ نسخة خام بامتداد `.realm` عند الحاجة.
- استيراد نسخة احتياطية على جهاز آخر من نفس صفحة الإعدادات.
- قبل الاستيراد، يقوم النظام تلقائيًا بحفظ نسخة حماية من قاعدة البيانات الحالية داخل مجلد البيانات.
- تعرض صفحة الإعدادات حجم قاعدة البيانات ومسارها وعدد السجلات الأساسية.

طريقة النقل لجهاز آخر:

1. من الجهاز الأول: الإعدادات → مدير النسخ الاحتياطي → إنشاء نسخة احتياطية.
2. انقل ملف `.mabackup` للجهاز الثاني.
3. من الجهاز الثاني: الإعدادات → مدير النسخ الاحتياطي → استيراد نسخة احتياطية.
4. بعد الاستيراد ستظهر البيانات على الجهاز الثاني.
